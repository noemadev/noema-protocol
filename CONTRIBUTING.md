# Contributing

Thank you for your interest in contributing to Noema Protocol.

This repository focuses on protocol specification,
architecture, and system design.

---

## Scope of Contributions

Contributions are welcome in the following areas:

- Documentation improvements
- Clarification of protocol concepts
- Architectural discussions
- RFC proposals for protocol changes
- Bug reports related to specifications

This repository does not accept:
- End-user applications
- Product-specific features
- Model-specific optimizations

---

## How to Contribute

### 1. Open an Issue
Before submitting a pull request, open an issue to:
- Report a problem
- Ask a question
- Propose a change
- Discuss an idea

This helps align expectations early.

---

### 2. RFCs for Major Changes
Significant or breaking changes require an RFC.

RFCs should clearly explain:
- The problem being solved
- The proposed design
- Alternatives considered
- Compatibility and impact

See the `rfcs/` directory for templates.

---

### 3. Pull Requests

When submitting a pull request:
- Keep changes focused and minimal
- Reference the related issue or RFC
- Use clear and simple language
- Avoid marketing or promotional content

Pull requests may be revised or closed
if they do not align with the project goals.

---

## Style Guidelines

- Prefer clarity over cleverness
- Prefer explicit definitions
- Avoid unnecessary abstractions
- Keep language neutral and technical

---

## Review Process

- Maintainers review all contributions
- Feedback may be requested
- Approval is at maintainer discretion
- Merged changes are documented in the changelog

---

## Code of Conduct

All contributors are expected to follow
the Code of Conduct defined in `CODE_OF_CONDUCT.md`.

---

## Notes

Noema Protocol values thoughtful contributions
that improve system understanding and design quality.
