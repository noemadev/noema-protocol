# Support

This document explains how to get support for Noema Protocol.

Noema Protocol is a protocol and specification project.
Support focuses on documentation, design discussion,
and clarification of intended behavior.

---

## Getting Help

For general questions or clarification:
- Use GitHub Issues
- Clearly describe what you are asking
- Provide relevant context or references

---

## Bug Reports

If you believe there is an issue with the protocol
specification or documentation:

- Open a GitHub Issue
- Describe the problem clearly
- Reference the relevant document or section

---

## Feature Requests and Proposals

Requests for new features or changes to the protocol
should be submitted as an RFC.

See the `rfcs/` directory for the RFC template
and contribution guidelines.

---

## Security Issues

Do NOT report security issues publicly.

For security-related concerns, follow the process
described in `SECURITY.md`.

---

## What Support Does Not Cover

Support does not include:
- End-user application troubleshooting
- Third-party implementations
- Deployment or infrastructure support
- Model-specific behavior

---

## Notes

Noema Protocol prioritizes clarity and correctness.
Support is provided on a best-effort basis during beta.
